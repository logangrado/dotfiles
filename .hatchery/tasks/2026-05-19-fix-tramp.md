# Task: fix-tramp

**Status**: complete
**Branch**: hatchery/fix-tramp
**Created**: 2026-05-19 21:39

## Objective

Opening remote projects over TRAMP was very slow and projectile indexing returned nothing.

## Context

Three interlocked root causes:

1. **No SSH connection reuse.** Every TRAMP op opened a fresh SSH handshake.
2. **vc-mode backend detection fired over TRAMP.** Each backend probe was a `file-exists-p` round-trip per file open.
3. **Projectile using `fd` instead of `git ls-files`.** Doom configures projectile to use `fd` as its indexing command even for git repos. `fd` is not installed on managed/embedded hosts like Home Assistant OS, so indexing silently returned empty.

## Summary

### Files changed

- `doom.d/packages/tramp.el`
- `doom.d/packages/projectile.el`

### tramp.el

- **SSH ControlMaster** (`ControlMaster=auto`, `ControlPersist=yes`) — one persistent socket per host, eliminating per-operation handshake cost.
- **`tramp-default-method "sshx"`** — sshx skips pty allocation, faster than ssh for TRAMP's non-interactive shell commands.
- **`vc-ignore-dir-regexp`** extended to skip vc-mode backend probing on TRAMP paths. Magit is unaffected (uses its own git process).
- **`tramp-own-remote-path`** added to `tramp-remote-path` so TRAMP inherits the remote user's full `$PATH` (needed for git and other tools to be findable).
- **LSP hook** commented out (was broken — `lsp-enable-lsp` doesn't exist). If LSP becomes a problem on remote files, use `(lsp-mode -1)` in the hook instead.
- **Mode-line lines** commented out — still useful but less critical with ControlMaster in place.

### projectile.el

- **`projectile-enable-caching t`** — session-scoped cache. With ControlMaster, reindexing at session start costs one `git ls-files` call over the warm socket. Persistent caching avoided in favour of always getting a fresh index.
- **`lg/projectile-remote-use-git-command` advice** on `projectile-get-ext-command`: for remote git projects, checks if `fd` is available on the remote host (`executable-find "fd" t`). Uses `fd` if present, falls back to `git ls-files -zco --exclude-standard` if not. Local projects keep Doom's `fd` default unmodified.

### Key gotchas

- `setq-local` in project hooks doesn't work for projectile — it reads `projectile-indexing-method` globally during indexing.
- TRAMP forbids reentrant calls. Cannot fall back to native traversal from inside an `:around` advice on `projectile-files-via-ext-command` — TRAMP is already mid-operation at that point.
- Doom's `doom--only-use-generic-command-a` advice on `projectile-get-ext-command` sets `fd` as the command even for git repos, overriding `projectile-git-command`. The fix must advise `projectile-get-ext-command` directly.
