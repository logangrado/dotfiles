# Task: fix-magit

**Status**: complete
**Branch**: hatchery/fix-magit
**Created**: 2026-05-01 08:36

## Objective

Fix two magit buffer management bugs related to worktrees and perspectives.

## Context

Bug 1 (pre-existing on branch): `lg/magit-kill-stale-log-buffers` killed the
active log buffer, corrupting `current-buffer` context and causing
`magit-log-setup-buffer` to resolve the wrong repo topdir when switching log
views across perspectives.

Bug 2 (new): after opening magit (`SPC g g`) in one worktree and switching to
a different worktree of the same project via RET, pressing `q` landed on the
original worktree's magit-status buffer instead of exiting magit. Both status
buffers belong to the same project but have different `default-directory`
values. `quit-window` restored the old one from the window history stack.

## Summary

### Design decision: one magit session per workspace

Rather than unwinding a stack of stale buffers on quit, the fix enforces an
invariant: **at most one magit-status buffer per persp-mode perspective**.
Stale buffers are never created in the first place.

### Changes (`doom.d/packages/magit.el`)

- **`lg/magit-kill-stale-log-buffers`**: added `(not (eq buf cur))` guard so
  the advice never kills the buffer it's called from. (Bug 1 fix.)

- **`lg/magit-status-for-workspace`** (`SPC g g`): now finds ANY
  `magit-status-mode` buffer in the perspective regardless of topdir. Goes to
  it and refreshes. Never opens a second one.

- **`lg/magit-cleanup-workspace-buffers`** (new, `:after` advice on
  `magit-status-setup-buffer`): when magit creates a new status buffer
  (e.g. RET on a worktree → `magit-worktree-status` →
  `magit-status-setup-buffer`), kills all other magit buffers in the
  perspective. The old status buffer is removed before it can pollute the
  window history.

- **`lg/magit--workspace-buffers`** and **`lg/magit-kill-workspace-magit-buffers`**:
  shared helpers extracted for reuse.

- Cleaned up debug artifacts in `lg/magit-log-branches` (`"test message"`
  string, unused `'magit-log-mode` arg) and removed commented-out delta
  color style args.
