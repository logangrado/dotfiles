# Task: rust-mode

**Status**: complete
**Branch**: hatchery/rust-mode
**Created**: 2026-06-08 15:33

## Objective

Install the rust tree-sitter grammar so opening `.rs` files no longer
raises `treesit (not-found)` warnings, and wire up full rust support
(LSP + grammar) in Doom Emacs.

## Context

`treesit-auto` was already configured in
`doom.d/packages/treesitter.el` with `(treesit-auto-install 'prompt)`,
but no rust-specific config existed and the Doom `rust` module was
disabled in `init.el`. Opening a `.rs` file triggered `rust-ts-mode`
(via `treesit-auto-add-to-auto-mode-alist`), which failed because
`libtree-sitter-rust.{so,dylib}` wasn't present in
`~/.emacs.d/.local/cache/tree-sitter/`. The expected prompt to
install the grammar evidently never fired or was declined.

## Summary

**Changes (commit `a67211d`):**

- `doom.d/init.el:170` — uncommented `(rust +lsp)` to enable the Doom
  rust module (rustic-mode, rust-analyzer LSP, cargo integration).
- `doom.d/packages/treesitter.el:8` — changed
  `(treesit-auto-install 'prompt)` → `(treesit-auto-install t)`.
  Future missing grammars now compile silently on first file open;
  no prompt to dismiss or miss.

**Follow-up the user runs locally (not in this worktree):**

1. `~/.emacs.d/bin/doom sync` to pull in the rust module's packages.
2. Restart Emacs (or `M-x doom/reload`).
3. Open any `.rs` file — `treesit-auto` will compile the grammar from
   source on first encounter (requires `cc` and `git`; both standard
   on macOS with `xcode-select`).
4. Optional: install `rust-analyzer` if `lsp-mode` doesn't offer to do
   it on first attach.

**Notes for future work:**

- `treesit-auto` ships recipes for ~all common languages, so adding
  more tree-sitter modes is now zero-touch — just open the file.
- Doom's rust module defaults to `rustic-mode` for `.rs` files, but
  `treesit-auto-add-to-auto-mode-alist` (in `treesitter.el`) remaps
  `.rs` to `rust-ts-mode` so the grammar is what actually runs. If
  rust-specific folding ranges become desirable, extend the
  `treesit-fold-range-alist` block in `treesitter.el` — the python
  setup there is the model to follow.
- The `lg/treesit-language-available-p` caching advice in
  `treesitter.el` means grammar-availability changes mid-session
  aren't seen until the cache is cleared (or Emacs restarted) — worth
  remembering if a future grammar install seems not to take effect.
