## Objective

Improve org-mode + org-roam system to make TODO capture, prioritization,
and ordering practical. He already has vulpea aggregating TODOs across roam notes;
the gaps were: too many TODO states, too many numeric priorities, no way to manually
order items within a priority bucket, `:project:` tag noise in agenda, and weekly
notes with no clear structure.

## Context

- Uses org-roam + vulpea: vulpea auto-tags any file with active TODOs with `:project:`,
  which powers `org-agenda-files` dynamically. Keep this pattern.
- TODOs stay scattered across roam notes (not centralized). Keep this too.
- Previous config: 5 numeric priorities (P0–P4), 9 active TODO states, no ordering.

### Key decisions

**TODO states** simplified to 6: `TODO → NEXT → PROG → WAIT | DONE / KILL`.
Face mapping: NEXT + PROG → `+org-todo-active` (cyan), WAIT → `+org-todo-onhold` (magenta).

**Priorities** A–D with org-fancy-priorities labels:
- `[#A]` → `HIGH` (red), `[#B]` → `MED` (orange), `[#C]` → `LOW` (yellow, default), `[#D]` → `BKG` (gray)

**Manual ordering** via `:ORDER:` integer property. New captures get `ORDER: 0`.
On every agenda open, `lg/org-agenda-normalize-orders` re-indexes items 1, 2, 3…
per priority group (ORDER=0 items float to top). J/K swap ORDER between adjacent
items within the same priority group and follow the moved item. Crossing group
boundaries is silently blocked. The normalization hook uses a `lg/org-agenda-normalize-pending`
flag to avoid infinite redo loops.

**Agenda column view** — custom prefix format shows `ORDER | FILE | STATE | ITEM | TAGS`.
File names have the roam timestamp prefix (`20250519152528-`) stripped and are
truncated to 15 chars. Priority badges (`[#X]`) are stripped from display via
`lg/agenda-strip-priority-cookies` in `org-agenda-finalize-hook` (redundant since
items are already grouped by priority).

**`:project:` tag** suppressed from agenda display via `org-agenda-hide-tags-regexp`.

**TODO capture** (`SPC n c t`) auto-inserts `ORDER: 0` so new items float to top.

**Meeting capture** (`SPC n c m`) — creates `meetings/<slug>.org` with `:meeting:`
filetag and a dated sub-heading. Uses `org-roam-capture-templates`.

**Weekly note skeleton** — `Goals / Log (Mon–Fri) / Carry-over` inserted on new file
creation. All three nav functions (`goto-today/next/prev`) use the same skeleton helper.

### Keybindings (all under `SPC n`)

| Binding | Action |
|---|---|
| `SPC n f` | Find roam node |
| `SPC n s` | Search roam |
| `SPC n a` | Ordered agenda |
| `SPC n c t` | Capture todo |

| `SPC n w w/n/p` | Weekly today/next/prev |
| `SPC n i` *(org)* | Insert roam link |
| `SPC n t` *(org)* | Set todo state |
| `SPC n p` *(org)* | Set priority |
| `SPC n g` *(org)* | Add roam tag |
| `j/k` *(agenda)* | Navigate items (skip headers) |
| `J/K` *(agenda)* | Move item down/up within priority group |

### Gotchas

- `org-super-agenda-mode` must be globally enabled for group local vars to work.
- `org-agenda-hide-tags-regexp "project"` hides any tag containing "project".
- J/K redo skips normalization by pre-setting `lg/org-agenda-normalize-pending t`.
- `(map! :leader "nc" nil)` is required before rebinding `SPC n c` as a prefix —
  Doom defaults it to `+org/toggle-last-clock`.
- evil normal state must be forced in org-agenda via `evil-set-initial-state` +
  `post-command-hook`; org-super-agenda headers reset evil state on cursor entry.
