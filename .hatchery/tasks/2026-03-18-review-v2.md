# Task: review-v2

**Status**: complete
**Branch**: hatchery/review-v2
**Created**: 2026-03-18 11:34

## Objective

Replace the bespoke `review.el` diff renderer with a magit-native system using
an isolated git worktree. The old system had no discard, slow staging (several
seconds per hunk due to delta re-render), and re-opened all hunks on refresh.

## Context

`review.el` built a custom two-section diff buffer on top of
`magit-section-mode`. `review-v2.el` replaces the custom renderer entirely with
real `magit-status` in a dedicated git worktree, so s/u/x (stage/unstage/discard)
are native magit at full speed with no redraw. REVIEW.md generation and inline
comments are adapted from review.el.

## Summary

### Files changed
- `doom.d/packages/review-v2.el` — new file, ~350 lines
- `.gitignore` — added `.hatchery/review-worktrees/`

### Key decisions

**Session key = git common dir** (`git rev-parse --git-common-dir`): the common
`.git` dir is the same value whether you call it from the main worktree or a
linked worktree. Keying by `(magit-toplevel)` would give different paths.

**Worktree at `.hatchery/review-worktrees/<branch-slug>/`**: consistent with
the existing `.hatchery/worktrees/` convention; gitignored.

**Git mechanics**: `git worktree add <path> <branch>` → `git reset --soft
<base>` (branch ref moves to base, all A…B changes staged) → `git restore
--staged .` (unstage into working tree). On publish: `git reset --hard
<old-sha>` (restores branch) → write + commit REVIEW.md → `git worktree
remove`.

**Comments in session plist**: unlike review.el which used buffer-local vars,
v2 stores comments under `:comments` in the session plist (keyed by repo key).
Overlays are re-applied via `magit-post-refresh-hook` using
`FILE\tHUNK-HEADER` as a stable section key.

**review.el untouched**: old `SPC g r d` (diff only) still works. The r/p/k/c
subkeys are overwritten by review-v2.el at load time (review.el loads first
alphabetically, review-v2.el overwrites).

### Gotchas

- During a review, `git log <branch>` from the main repo shows the branch
  pointing to `<base>`, not the original HEAD. This is expected — the branch
  is temporarily at `<base>` until publish restores it with `git reset --hard
  old-sha`.

- `magit-status-setup-buffer` is called with `default-directory` bound to the
  worktree path. It displays the buffer itself (no separate `switch-to-buffer`
  needed).

- `lg/review--hunk-line-at-point` and `lg/review--grab-diff-context` (from
  review.el) are reused directly. `lg/review--parsed-files` won't be set in
  the magit-status buffer, so `grab-diff-context` always uses its fallback
  path (reads from buffer), which is fine.

- `lg/review-instructions` (defined in review.el) is referenced directly in
  review-v2.el — safe because review.el loads first.
