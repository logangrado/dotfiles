# Task: vterm-dim

**Status**: complete
**Branch**: hatchery/vterm-dim
**Created**: 2026-05-01 10:50

## Objective

for most buffers, when we switch away from them, the background dims. The focused buffer has a brighter background.

This is not true for our vterm buffers.

Can we fix this?

## Summary

**Problem:** `auto-dim-other-buffers` uses `:filtered (:window ...)` face specs
to dim unfocused buffers. This mechanism operates at Emacs display time, but
vterm's C-level libvterm rendering resolves face colors before that filtering
kicks in, so vterm buffers never visually dim.

**Solution:** Added a `window-selection-change-functions` hook in the vterm
config (`doom.d/packages/vterm.el`, lines 139-175) that applies a direct
(non-filtered) `face-remap-add-relative` on the `default` face for unfocused
vterm buffers. It reads the background color from the `auto-dim-other-buffers`
face so the colors match exactly. `vterm--invalidate` is called after each
change to force libvterm to re-render.

**Key functions added:**
- `lg/vterm-dim--apply` / `lg/vterm-dim--remove` — buffer-local dim/undim
- `lg/vterm-dim--on-selection-change` — iterates all visible windows on focus change

**Gotchas:**
- The face remap only affects cells using the `default` face (terminal default
  background). Cells with explicit vterm-color-* faces (colored terminal
  backgrounds) won't be dimmed — this matches how auto-dim behaves for
  syntax-highlighted text in normal buffers.
- If the theme changes at runtime, the dimmed color is re-read on each
  apply call, so it stays in sync.
